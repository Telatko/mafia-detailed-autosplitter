Mafia Autosplitter Setup

To set up an autosplitter for Mafia speedruns you need to get either

  1. the official per-mission one, which is easy to set up,
  2. the per-checkpoint one, which is slightly more difficult to set up, or
  3. the fine-grained one you'll find out how to set up if you keep reading.

Fine-grained Mafia autosplitter

The newest version of the autosplitter allows you to split not just per checkpoint but also at certain events during your run.

If you haven't already download the autosplitter script. It doesn't matter where you store the file.

Using pre-made splits and layouts

If you aren't too picky when it comes to which events should trigger splits and you are okay with using pre-made splits then setting up the splitter is easy. Please note that these are suitable only for 1.0 version of the game.

  1. Download your split and layout file from the tables below depending on the category you want to speedrun and how detailed the splits should be
  2. Open the split (.lss) file you just downloaded in LiveSplit.
  3. Right click the splits, Open Layout -> From File... and select the layout (.lsl) file you just downloaded.
  4. Right click the splits again, Edit Layout, double click Scriptable Auto Splitter. There you need to click the Browse button and select the auto splitter script (.asl) file. Close both windows when you're done.
  5. Right click the splits again and select Save Layout to make sure you won't lose the changes you just made.
  6. Right click the splits again and select Edit Splits.... If there is a button Deactivate then click it so that it says Activate. This disables the default Mafia autosplitter. If you don't do this there will be one extra split at the beginning of each mission.

Progress splits

You might have noticed there are so-called Progress splits in the tables above. They are not meant to be a replacement for your primary splits but rather an interesting addition. Basically you would use Realistic SoB or Detailed splits as your primary splits and Progress splits in a separate LiveSplit window. To setup Progress splits simply repeat the same steps above.

Once you finish at least one run Progress splits can show you live prediction of your final time. The predicted time would change all the time without the need to finish a segment in your primary splits. Say you are driving somewhere and you need to slow down because of cops. The moment you start slowing down the predicted time would start increasing. This works by splitting all usual routes into many pieces, each having its own segment in LiveSplit. For this reason you should not show the splits in your layout.

Custom split events

The steps described above are all you need to do to successfully set up your splits. But if you want to selectively choose which events should trigger splits to suit your needs better then you can do that too. This option is more complicated though. In particular, you need to

  1. Select the autosplitter (.asl) file using the Choose File button at the top of the page. Split tables should appear where you can select which split events you want to get triggered and what names the splits should use.

  2. Split tables can be saved and loaded to and from json files using the Choose File and Save buttons above.

    You can also load pre-existing split tables to save some time:

    • Vanilla* Any% Detailed,
    • Vanilla* Bertone% Detailed,
    • Vanilla* MasterRace% Detailed,
    • Vanilla* Realistic SoB (both Any% and Bertone%),
    • Speedrun Mod Any% Detailed,
    • Speedrun Mod Bertone% Detailed,
    • Speedrun Mod Realistic SoB (both Any% and Bertone%),
    • Chaos Mod Any% Detailed,
    • Vanilla* Any% Progress
    • Vanilla* Bertone% Progress

    * Official version of the game or OHKO mod

    Runners are encouraged to share their custom split tables.

  3. Once you are happy with the split tables you can generate your split file using the Generate split file button. Use any of the Generate IL Split File buttons to generate an IL split file.

  4. Prepare a layout file. In particular

    • Open the generated split file in livesplit, right click it and select Edit Layout....
    • If you use the Splits component drop it and replace it with the List->Subsplits one.
    • Add the Control->Scriptable Auto Splitter component, edit it and select the autosplitter file you downloaded previously.
    • Close both the Layout Settings and Layout Editor windows by clicking the OK button.
    • Right click you splits, select Save Layout As... and store the layout file somewhere.

    You can skip this step by using a layout file that already has both the Subsplits and Scriptable Auto Splitter components. For this purpose you can use any of the pre-made layout files above.

  5. Select the layout file using the Choose File button and and then click the Save button to save your split table settings into your layout.
  6. Right click your splits in livesplit, select Open Layout -> From File... and select the file you just saved.
  7. Right click the splits again and select Edit Splits.... If there is a button Deactivate then click it so that it says Activate. This disables the default Mafia autosplitter. If you don't do this there will be one extra split at the beginning of each mission.

Please note these steps need to be repeated everytime you want to make any changes to your splits.

So why do we need those split tables? Why not just name our splits using the Splits Editor in livesplit and select our split events within the autosplitter settings?

You can indeed do that. The only issue is that since this is not the official autosplitter for Mafia we have to use the Scriptable Auto Splitter component in our layout instead of setting it within our split file. This leads to our settings ending up getting saved in the layout instead of the split file. So we would have two different files we would need to keep in sync. Enabling or disabling any split event within the autosplitter settings would require changing your split file too. Split tables allow us to make changes in one place and then regenerate our split and layout files.
